Android应用程序Window的内部机制

Posted on 2018-12-22

Window机制分析

Android中每一个视图都是一个Window,同时每一个Window都对应着一个View和一个ViewRootImpl(这个类是最重要的桥梁).Window是一个抽象类，它的具体实现是PhoneWindow.通常我们进行开发时主要是通过WindowManager来对Window进行操作.Window的具体实现是在WindowManagerService中。

从以上的代码中我们可以知道通过WindowManager就可以操作一个Window.

1.Window的添加过程:

WindowManager是一个接口它继承了ViewManager

public interface WindowManager extends ViewManager

public interface ViewManager
{
/**
 * Assign the passed LayoutParams to the passed View and add the view to the window.
 * <p>Throws {@link android.view.WindowManager.BadTokenException} for certain programming
 * errors, such as adding a second view to a window without removing the first view.
 * <p>Throws {@link android.view.WindowManager.InvalidDisplayException} if the window is on a
 * secondary {@link Display} and the specified display can't be found
 * (see {@link android.app.Presentation}).
 * @param view The view to be added to this window.
 * @param params The LayoutParams to assign to view.
 */
public void addView(View view, ViewGroup.LayoutParams params);
public void updateViewLayout(View view, ViewGroup.LayoutParams params);
public void removeView(View view);
}`

在ViewManager中有三个重要的方法：

  • addView：添加View
  • updateViewLayout:更新View
  • removeView:移除View
    WindowManager的具体实现是WindowManagerImpl类，在WindowManagerImpl它的具体实现如下:

从上面的代码中可以看出WindowManagerImpl又将View的操作委托给了WindowManagerGlobal。在WindowManagerGlobal的addView内部创建了

ViewRootImpl root = new ViewRootImpl(view.getContext(), display);

然后继续调用ViewRootImpl中的setView方法:

在setView方法中首先首先更新视图requestLayout();然后通过WindowSession来完成视图的添加,在Session内部最后通过WindowManagerService来完成View的添加，最终Window视图的添加就交给了WindowManagerService来处理了.

Window的删除过程

Window的删除过程也是通过WindowManagerGlobal的操作完成.

WindowManagerGlobal的具体删除操作代码:

首先通过findViewLocked来索引出待删除的View,然后调用removeViewLocked：

通过索引获取到ViewRootImpl来完成删除操作。具体的操作主要是由ViewRootImpl的die方法发送删除请求然后返回一个boolean类型。

die方法只是发送了一个请求删除的消息，这个时候View并没有完成删除操作。
ViewRootImpl会调用handle和doDie方法。在doDie内部调用dispatchDetachedFromWindow是Window真正的删除操作.

最终会执行mWindowSession.remove(mWindow);删除这个View。

Window的更新过程

Window的更新逻辑最终还是调用WindowManagerGlobal:

updateViewLayout首先需要更新View的LayoutParams,然后调用ViewRootImpl的setLayoutParams方法来更新视图。

最终通过WindowManagerService的relayoutWindow来更新视图。
